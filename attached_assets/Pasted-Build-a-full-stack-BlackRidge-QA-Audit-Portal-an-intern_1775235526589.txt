Build a full-stack BlackRidge QA Audit Portal — an internal tool for BlackRidge Platforms that lets authenticated users run automated quality assurance checks on any website or API URL and view the results in a live dashboard.




Tech Stack

Use the following stack:

•
Backend: Python with FastAPI

•
Frontend: React with Vite (or a single HTML/JS/CSS file if simpler in Replit)

•
Database: SQLite (via SQLAlchemy or raw sqlite3) for storing audit history

•
AI Analysis: OpenAI API (gpt-4o-mini) for generating executive summaries

•
Styling: Tailwind CSS via CDN, dark theme

•
Real-time progress: Server-Sent Events (SSE) from FastAPI to the frontend

Store the OpenAI API key as a Replit Secret named OPENAI_API_KEY.

QA Agent Modules

Each agent must be a Python class that accepts a target_url and optional auth_token, runs its tests, and returns a list of findings. Each finding is a dict with: agent, test_name, status, severity, title, description, evidence, remediation, endpoint, response_code, response_time_ms.

Security Agent (agents/security_agent.py)

Tests to implement (each as a separate method):

1.
Security Headers — Check for X-Frame-Options, X-Content-Type-Options, Strict-Transport-Security, Content-Security-Policy, X-XSS-Protection. Mark MISSING headers as HIGH severity findings.

2.
HTTPS Enforcement — If URL is HTTP, flag as HIGH. If HTTPS, verify it doesn't redirect to HTTP.

3.
SQL Injection — Append ' OR '1'='1, ; DROP TABLE users;--, 1 OR 1=1 to query params on known endpoints. If response contains SQL error keywords (syntax error, mysql, ORA-, pg_), flag as CRITICAL.

4.
Cross-Site Scripting (XSS) — Inject <script>alert(1)</script> and "><img src=x onerror=alert(1)> into query params. If the payload is reflected unescaped in the response body, flag as HIGH.

5.
Authentication Bypass — Try accessing /api/users, /api/admin, /api/dashboard, /api/profile without any auth token. If response is 200 (not 401/403), flag as HIGH.

6.
Sensitive Data Exposure — Scan response bodies for patterns matching API keys (sk-, AKIA, Bearer ), passwords ("password":, "secret":), or private keys (-----BEGIN). Flag as CRITICAL if found.

7.
Dangerous HTTP Methods — Send OPTIONS request and check if TRACE or TRACK is listed in Allow header. Flag as MEDIUM if present.

8.
Rate Limiting — Send 15 rapid POST requests to /api/auth/login or /api/login. If all return non-429 status, flag missing rate limiting as HIGH.

9.
CORS Misconfiguration — Send request with Origin: https://evil.blackridge-qa-test.com. If Access-Control-Allow-Origin reflects back the evil origin or is * with credentials, flag as CRITICAL.

Infrastructure Agent (agents/infrastructure_agent.py )

Tests to implement:

1.
API Reachability — Simple GET to base URL. If connection refused or timeout, flag as CRITICAL.

2.
Response Time — Measure response time. Flag as WARNING if >2000ms, FAILED if >5000ms.

3.
HTTPS/SSL — Verify SSL certificate is valid and not expired using Python's ssl module.

4.
CORS Headers Present — Verify CORS headers exist on API responses.

5.
Error Handling — Request a non-existent endpoint like /api/this-does-not-exist-qa-test. Verify it returns 404 (not 200 or 500). Check that the error response does not contain stack traces.

6.
Large Payload Handling — Send a POST with a 10MB body. Verify server returns 413 or similar (not a crash/500).

7.
Concurrent Load — Send 10 simultaneous GET requests. Verify all return non-500 status codes.

8.
Health Endpoint — Check if /health, /api/health, or /ping exists and returns 200.

9.
HTTP to HTTPS Redirect — If target is HTTPS, verify that the HTTP version redirects to HTTPS.

API Agent (agents/api_agent.py)

Tests to implement:

1.
Content-Type Headers — Verify all API responses include Content-Type: application/json.

2.
HTTP Method Enforcement — Try GET on POST-only endpoints and vice versa. Verify 405 is returned.

3.
Input Validation — Empty Body — POST to known endpoints with an empty {} body. Verify 400/422 is returned, not 500.

4.
Input Validation — Invalid Email — Submit not-an-email in email fields. Verify 400/422 is returned.

5.
Input Validation — Negative Numbers — Submit -999 for price/amount fields. Verify validation rejects it.

6.
Consistent Error Format — Check that all error responses follow a consistent structure (e.g., always have error or message field).

7.
Pagination on List Endpoints — Check that list endpoints (/api/users, /api/orders) return pagination metadata (page, total, limit).

8.
Idempotency of GET — Call the same GET endpoint 3 times. Verify identical responses.

9.
API Versioning — Check if the API uses versioning (/api/v1/, /v1/, Accept: application/vnd.api+json). Flag as LOW if no versioning detected.

Data Flow Agent (agents/data_flow_agent.py)

Tests to implement:

1.
Create-Read Consistency — POST to create a resource, then GET it back. Verify submitted fields match retrieved fields. Clean up after test.

2.
Data Type Integrity — POST a payload with mixed types (int, float, bool, string, null). Verify types are preserved in the response.

3.
Required Field Validation — PATCH an existing resource with null values for required fields. Verify 400/422 is returned.

4.
Update Propagation — PATCH a resource, then immediately GET it. Verify the update is visible.

5.
Delete Verification — DELETE a resource, then GET it. Verify 404 is returned.

6.
Financial Calculation — POST to an order calculation endpoint with known values. Verify subtotals, taxes, and totals are mathematically correct to 2 decimal places.

7.
Data Sanitization — POST <script>alert(1)</script> as a field value. Verify it is sanitized in the response.

8.
Timestamp Format — Check that all timestamp fields in responses use ISO 8601 format (YYYY-MM-DDTHH:MM:SSZ).

9.
Concurrent Write Safety — Send 5 simultaneous PATCH requests to the same resource. Verify no 500 errors occur.




QA Orchestrator (qa_runner.py)

The orchestrator coordinates all four agents:

Python


class QAOrchestrator:
    def __init__(self, audit_id, target_url, auth_token, known_endpoints, db_update_callback, progress_callback):
        # progress_callback(event_type, data) pushes SSE events
        # db_update_callback(audit_id, updates) writes to DB
        pass

    def run(self):
        agents = [
            ("Security", SecurityAgent),
            ("Infrastructure", InfrastructureAgent),
            ("API", APIAgent),
            ("Data Flow", DataFlowAgent),
        ]
        all_findings = []
        for agent_name, AgentClass in agents:
            self.progress_callback("progress", {"agent": agent_name, "message": f"Running {agent_name} tests..."})
            self.db_update_callback(self.audit_id, {"status": "running", "current_agent": agent_name})
            agent = AgentClass(self.target_url, self.auth_token, self.known_endpoints)
            findings = agent.run()
            all_findings.extend(findings)
            passed = len([f for f in findings if f["status"] == "PASSED"])
            failed = len([f for f in findings if f["status"] == "FAILED"])
            self.progress_callback("agent_complete", {"agent": agent_name, "passed": passed, "failed": failed})

        # Calculate score
        score = self._calculate_score(all_findings)
        grade = self._score_to_grade(score)

        # Generate AI analysis using OpenAI
        ai_analysis = self._generate_ai_analysis(all_findings, score)

        # Generate markdown report
        report_markdown = self._generate_markdown_report(all_findings, score, grade, ai_analysis)

        # Save everything to DB
        self.db_update_callback(self.audit_id, {
            "status": "completed",
            "score": score,
            "grade": grade,
            "ai_analysis": ai_analysis,
            "report_markdown": report_markdown,
            "completed_at": datetime.utcnow().isoformat(),
            ...
        })

        self.progress_callback("complete", {"score": score, "grade": grade, "audit_id": self.audit_id})



Score calculation:

•

•
Each CRITICAL finding: -25 points

•
Each HIGH finding: -10 points

•
Each MEDIUM finding: -5 points

•
Each LOW finding: -2 points

•
Minimum score: 0

Grade mapping:

•
95–100 → A+

•
90–94 → A

•
85–89 → B+

•
80–84 → B

•
70–79 → C

•
60–69 → D

•
0–59 → F

Real-Time Progress Implementation

Use Server-Sent Events (SSE):

Python


# FastAPI SSE endpoint
from fastapi.responses import StreamingResponse
import asyncio, json

@app.get("/api/audits/{audit_id}/stream")
async def stream_audit(audit_id: int):
    async def event_generator():
        while True:
            event = await get_next_event(audit_id)  # poll from a queue
            if event is None:
                break
            yield f"data: {json.dumps(event)}\n\n"
            if event["type"] in ["complete", "error"]:
                break
    return StreamingResponse(event_generator(), media_type="text/event-stream")



On the frontend, use the native EventSource API:

JavaScript


const es = new EventSource(`/api/audits/${auditId}/stream`);
es.onmessage = (e) => {
  const event = JSON.parse(e.data);
  if (event.type === 'progress') updateAgentStatus(event.agent, 'running', event.message);
  if (event.type === 'agent_complete') updateAgentStatus(event.agent, 'done', `${event.passed} passed, ${event.failed} failed`);
  if (event.type === 'complete') { es.close(); loadAuditReport(event.audit_id); }
  if (event.type === 'error') { es.close(); showError(event.message); }
};

Important Notes for Replit Agent

1.
All HTTP requests in the QA agents must use requests with a timeout of timeout=10 to prevent hanging.

2.
Wrap every individual test in a try/except block — if a test throws an exception, record it as a SKIPPED finding with the error message, never crash the agent.

3.
The SSE streaming must use an asyncio.Queue per audit_id stored in a global dict. The background thread puts events into the queue; the async SSE endpoint reads from it.

4.
Use threading.Thread(target=orchestrator.run, daemon=True) to run the QA orchestrator in the background without blocking FastAPI.

5.
Store the OpenAI API key as os.environ.get("OPENAI_API_KEY"). If not set, skip AI analysis and use a rule-based summary instead.

6.
The frontend must handle the case where the SSE connection drops (network error) by polling GET /api/audits/{audit_id} every 3 seconds as a fallback.

7.
Make the UI fully responsive — it must work on mobile screens (single column layout below 768px).

8.
Add a "Copy URL" button next to each finding's endpoint field for quick access.

9.
The audit history must persist across server restarts (SQLite handles this automatically).

10.
Set Access-Control-Allow-Origin: * on all API responses during development.
