speaking almost immediately after I finish talking, not wait for the 
full response to generate first.

new streaming route: POST /api/ridge/stream

This route should:

1. Accept { messages } in the request body — clean them the same way 
   the current chat route does (role mapping, alternating check)

2. Set response headers for streaming:
   res.setHeader('Content-Type', 'audio/mpeg')
   res.setHeader('Transfer-Encoding', 'chunked')
   res.setHeader('Cache-Control', 'no-store')

3. Call the Anthropic API with stream: true using this pattern:
   const anthropicResponse = await fetch('https://api.anthropic.com/v1/messages', {
     method: 'POST',

       'Content-Type': 'application/json',
       'x-api-key': process.env.ANTHROPIC_API_KEY,
       'anthropic-version': '2023-06-01'
     },
     body: JSON.stringify({
       model: 'claude-sonnet-4-20250514',
       max_tokens: 1000,
       stream: true,
       system: RIDGE_SYSTEM,
       messages: trimmed.slice(-14)
     })
   });

4. Read the Anthropic stream and accumulate text chunks into a buffer. 
   When the buffer reaches 120 characters AND ends with a sentence-ending 
   punctuation mark (period, question mark, exclamation mark, or colon), 
   flush that buffer to ElevenLabs immediately as a TTS chunk and reset 
   the buffer. This way RIDGE starts speaking the first sentence while 
   still generating the rest.
