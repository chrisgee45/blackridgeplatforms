Add a Client Discovery feature inside the existing Projects section of the BlackRidge Platforms portal. This is not a standalone page — it lives within a project's detail view. When I'm working a project, I can send that client a discovery form directly from their project, and all responses come back and live on that project record.

TECH STACK

React + Tailwind (existing portal stack)
Supabase for data storage
Resend for email delivery
Supabase Storage for file uploads
Existing portal auth context and project data structure


SUPABASE TABLES — create if they don't exist
discovery_submissions

id (uuid, primary key)
project_id (uuid, foreign key → projects.id)
client_name (text)
client_email (text)
company_name (text)
industry (text)
referral_source (text)
token (uuid, unique)
status (text) — "sent" | "opened" | "submitted"
sent_at (timestamp)
submitted_at (timestamp)
responses (jsonb)
uploaded_files (jsonb) — array of file metadata: name, url, size, type
signature_acknowledged (boolean)
notes (text)
created_at (timestamp)


WHERE IT LIVES IN THE UI
Inside the existing project detail view, add a new tab called "Discovery" alongside existing tabs.
Discovery Tab — before a form has been sent:

Clean empty state: "Send your client a discovery form to capture their goals, priorities, and expectations before the project kicks off."
Button: "Send Discovery Form" (gold)
Opens a small modal to confirm or edit: Client Name, Client Email, Company Name (pre-filled from project data if available)
On confirm: generate UUID token, insert row into discovery_submissions linked to this project_id, send email via Resend, show success toast: "Discovery form sent to [email]"

Discovery Tab — after form is sent, not yet submitted:

Status card: client name, email, status badge (gray = sent, yellow = opened), sent timestamp
"Resend Link" button — resends the same email with the existing token
Copy link icon to grab the form URL manually

Discovery Tab — after client submits:

Status badge turns green: "Submitted"
Show submitted timestamp
All responses displayed in clean read-only layout organized by section
Any uploaded files shown as downloadable links with file name, type, and size
Internal notes textarea I can type into and save
"Start Proposal" button placeholder for future workflow connection


PUBLIC CLIENT FORM /discovery/:token
Accessible without auth. On load, look up the token. If invalid or already submitted, show a polite error. If valid, mark status "opened" and render the form.
