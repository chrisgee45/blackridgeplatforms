Add a Client Kickoff Discovery feature inside the existing Projects section of the BlackRidge Platforms portal. This lives within a project's detail view. Once a client commits, I send them this form to collect everything I need before I touch a single line of code. All responses, files, and answers come back and live on that project record inside the portal.

TECH STACK

React + Tailwind (existing portal stack)
Supabase for data storage
Resend for email delivery
Supabase Storage for file uploads
Existing portal auth context and project data structure


SUPABASE TABLES — create if they don't exist
kickoff_submissions

id (uuid, primary key)
project_id (uuid, foreign key → projects.id)
client_name (text)
client_email (text)
company_name (text)
token (uuid, unique)
status (text) — "sent" | "opened" | "submitted"
sent_at (timestamp)
submitted_at (timestamp)
responses (jsonb)
uploaded_files (jsonb) — array of file metadata: name, url, size, type
signature_acknowledged (boolean)
notes (text)
created_at (timestamp)


WHERE IT LIVES IN THE UI
Inside the existing project detail view, add a tab called "Kickoff" alongside existing tabs.
Kickoff Tab — before form is sent:

Empty state: "Send your client the kickoff form to collect everything you need before the build starts."
Button: "Send Kickoff Form" (gold)
Opens a small modal to confirm or edit: Client Name, Client Email, Company Name (pre-filled from project data)
On confirm: generate UUID token, insert row into kickoff_submissions linked to project_id, send email via Resend, toast: "Kickoff form sent to [email]"

Kickoff Tab — sent, not yet submitted:

Status card showing client name, email, status badge (gray = sent, yellow = opened), sent timestamp
"Resend Link" button
Copy link icon for manual sharing

Kickoff Tab — submitted:

Status badge turns green: "Submitted"
Submitted timestamp
All responses in clean read-only layout organized by section
Uploaded files shown as downloadable links with file name, type icon, and size
Internal notes textarea I can save back to the record
"Build Proposal" placeholder button for future connection


PUBLIC CLIENT FORM /kickoff/:token
No auth required. On load, validate the token. Invalid or already submitted shows a polite error. Valid token marks status "opened" and renders the form.
