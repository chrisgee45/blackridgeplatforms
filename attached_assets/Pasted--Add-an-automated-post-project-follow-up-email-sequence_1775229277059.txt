
Add an automated post-project follow-up email sequence to the BlackRidge Platforms portal that triggers when a project is marked Completed. This sequence runs over 12 months and is designed to retain clients, generate referrals, and create repeat business. It follows the same pattern as the welcome sequence — automatic trigger on status change plus a manual trigger for existing completed projects.

SUPABASE TABLE — create if it doesn't exist
followup_sequences

id (uuid, primary key)
project_id (uuid, foreign key → projects.id)
client_name (text)
client_email (text)
company_name (text)
status (text) — "running" | "completed" | "cancelled"
email_1_sent_at (timestamp) — thank you + referral
email_2_sent_at (timestamp) — 30 day check in
email_3_sent_at (timestamp) — 90 day touch base
email_4_sent_at (timestamp) — 6 month check in
email_5_sent_at (timestamp) — 1 year anniversary
triggered_by (text) — "auto" | "manual"
created_at (timestamp)


TRIGGER 1 — AUTOMATIC
When a project's status is changed to "Completed":

Check if a follow-up sequence already exists for this project — if running or completed, do nothing
If none exists, create a row in followup_sequences and start the sequence immediately
Toast: "Follow-up sequence started for [Client Name]"


TRIGGER 2 — MANUAL
Inside the project detail view, inside the existing "Sequence" tab, add a second section below the welcome sequence called "Follow-Up Sequence".
Before sequence starts:

Label: "Post-Project Follow-Up"
Description: "Start the follow-up series to thank your client, request referrals, and stay top of mind over the next 12 months."
Button: "Start Follow-Up Sequence" (gold)
Confirmation modal pre-filled with client name, email, company — editable
On confirm: fire Email 1 immediately, schedule the rest, toast: "Follow-up sequence started — thank you email sent to [email]"

While running:

Same timeline view as the welcome sequence
Each email shows: subject, status (sent / scheduled / pending), sent timestamp or scheduled date
"Cancel Sequence" button with confirmation
"Resend" option on any sent email

Completed:

All five checkmarks green
Final sent timestamp
"Restart Sequence" option if needed

IMPORTANT LOGIC NOTES

Scheduled emails must survive server restarts — use Supabase Edge Functions with a cron job that checks for due emails every hour and fires any that are past their scheduled time
Store the scheduled send time for each email in the followup_sequences row so the Sequence tab can display "Scheduled for [date]" on pending emails
If a project is reopened or reactivated after being marked complete, do NOT cancel the follow-up sequence — it should keep running
Only one follow-up sequence per project — manual trigger shows "Restart" if one already exists
All send attempts logged with success or failure status visible in the Sequence tab
Existing completed projects: manual trigger only, no auto-trigger on deploy


SEQUENCE TAB — FULL PICTURE
The Sequence tab inside each project now shows two sections stacked:

Welcome Sequence — 3 emails, same-day, fires on Active
Follow-Up Sequence — 5 emails, 12 months, fires on Completed

Both sections follow the same timeline UI. At a glance you can see exactly where every client stands in their journey with BlackRidge — from the day they commit to a full year after you deliver.
